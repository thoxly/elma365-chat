# Шаблоны заданий (Task Templates)

## Назначение

Шаблон заданий описывает, как ассистент должен обрабатывать запрос пользователя: какой промпт и системный промпт использовать, какие MCP-инструменты разрешить, какие правила из БД подставить.

## Поля шаблона

- **name** — название (например «Валидация процесса»).
- **description** — описание (опционально).
- **prompt** — промпт для LLM; может содержать плейсхолдеры `{user_input}` и `{context}`.
- **system_prompt** — системный промпт (роль, ограничения).
- **tools** — список имён MCP-инструментов (например `["elma365.search_docs", "elma365.get_doc"]`).
- **knowledge_rules** — список типов правил из таблицы `knowledge_rules` (например `["architecture_rules", "process_rules"]`), которые подставляются в контекст.

## Использование

При отправке сообщения в чат можно указать `template_id`. Тогда FlexibleAgent будет использовать этот шаблон для формирования запроса к LLM и вызова MCP.

## API

CRUD: `GET/POST/PUT/DELETE /api/templates/` и `/api/templates/{id}`. См. [docs/api.md](../docs/api.md).
