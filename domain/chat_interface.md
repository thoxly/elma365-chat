# Чат-интерфейс

## Назначение

Чат позволяет пользователю общаться с ассистентом по документации ELMA365. Сообщения сохраняются в БД (таблица `chat_messages`). Контекст может включать загруженные документы (таблица `chat_documents`) и историю диалога.

## Гибкий агент

Ответы генерирует **FlexibleAgent** ([app/services/flexible_agent.py](../app/services/flexible_agent.py)):

- Работает по **шаблону задания** (TaskTemplate): промпт, системный промпт, список MCP-инструментов, список правил из БД.
- Загружает правила ELMA365 из таблицы `knowledge_rules` и подставляет их в системный промпт.
- Вызывает LLM (DeepSeek по умолчанию) с возможностью вызова MCP-инструментов (поиск по документации и т.д.).

Если шаблон не выбран, чат возвращает простое эхо или подсказку использовать шаблон.

## Сессии

- Идентификаторы: `user_id`, `session_id`. На фронте по умолчанию используются константы (например `web-user`, `default-session`). В будущем можно привязать к авторизации.

## API

- Отправка сообщения: `POST /api/chat/messages` (user_id, session_id, message, template_id?).
- История: `GET /api/chat/sessions/{session_id}/history?user_id=`.
- Загрузка документа: `POST /api/chat/documents` (form: user_id, session_id, file).

См. [docs/api.md](../docs/api.md).
